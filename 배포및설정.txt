아이온2 성역 일정 사이트 — GitHub Pages + Supabase 설정

1) Supabase (무료)
   - https://supabase.com 에서 프로젝트 생성
   - SQL Editor에서 supabase/schema.sql 내용 실행
   - Authentication → Providers → Anonymous sign-ins 켜기
   - Project Settings → API 에서 URL / anon public key 복사

2) 로컬 개발
   - 프로젝트 루트에 .env 파일 생성:

     VITE_SUPABASE_URL=https://xxxx.supabase.co
     VITE_SUPABASE_ANON_KEY=eyJhbGciOi...

   - npm install
   - npm run dev

3) GitHub Pages (Actions 빌드)
   - 저장소 Settings → Secrets and variables → Actions
     - VITE_SUPABASE_URL
     - VITE_SUPABASE_ANON_KEY
     - (선택) VITE_BASE
       * 기본값: /<저장소이름>/  (예: https://user.github.io/repo-name/)
       * <계정>.github.io 의 루트 저장소(사용자 페이지)처럼 경로가 '/' 여야 하면 VITE_BASE 를 "/" 로 지정
   - Settings → Pages → Build and deployment → Source: GitHub Actions
   - main(또는 master)에 푸시하면 자동 배포
   - 사이트 주소: https://<계정>.github.io/<저장소이름>/

4) 정적 호스팅만으로 한계
   - 브라우저만으로 안전하게 “누구나 쓰기”를 열려면 백엔드가 필요합니다.
   - 이 프로젝트는 Supabase 익명 로그인으로 계정 UI 없이 세션만 만들고,
     닉네임/서버는 표시용으로만 저장합니다.

5) 전투력·템렙 자동 갱신(Edge Function) — `fetch-combat-power`
   - 「목록·전투력 갱신」은 본인 행이 있으면 Edge가 플레이NC **공식 검색 JSON API**(
     `/ko-kr/api/search/aion2/search/v2/character` )로 `characterId`를 찾고, 프로필 URL(
     `/ko-kr/characters/{serverId}/{characterId}` )의 HTML에서 전투력·템렙을 읽습니다.
   - 검색은 마족(race=2) → 천족(race=1) → race 생략 순으로 시도합니다. **닉네임은 API 응답의 이름(HTML 태그
     제거 후)과 정확히 일치**해야 합니다.
   - 프로필 **초기 HTML에 전투력 블록이 없으면**(공식이 브라우저에서만 그리는 경우) 자동 반영은 실패하고,
     응답에 프로필 URL이 포함될 수 있습니다. 그때는「전투력 반영」으로 수동 입력하세요.
   - 배포 절차 (저장소 루트에서, Node/npm 이 있으면 CLI 전역 설치 없이 npx 만으로 가능):
       npx supabase@latest login
       npx supabase@latest link --project-ref <Project Reference ID>
       npx supabase@latest functions deploy fetch-combat-power
     ※ Project Reference ID: 대시보드 → Project Settings → General → Reference ID
       (보통 API URL https://<이값>.supabase.co 의 서브도메인과 동일합니다.)
     ※ URL 전체를 넣지 마세요. 예: https://abcd.supabase.co → --project-ref abcd
     ※ 배포 후: 대시보드 왼쪽 **Edge Functions** 에 `fetch-combat-power` 가 목록에 보여야 합니다.
   - npm 스크립트: 프로젝트를 이미 link 해 두었다면
       npm run deploy:functions
   - 그래도 같은 오류면: 브라우저 **광고 차단**·**회사/학교망**이 supabase 도메인을 막는지,
     `.env` / GitHub Secrets 의 `VITE_SUPABASE_URL` 이 **배포한 프로젝트와 동일한지** 확인합니다.
   - 대시보드에서 **POST 401** 이고 본문에 **Unsupported JWT algorithm ES256** 이면, Auth JWT 가 ES256 인데
     Edge 게이트 검증이 막는 경우입니다. 저장소의 `supabase/config.toml` 에서 해당 함수 `verify_jwt = false` 로
     두었으니 **한 번 더** `npm run deploy:functions` 하면 반영됩니다. (인증은 함수 코드의 getUser() 로 처리)
   - 대시보드에 함수가 있는데도 **「non-2xx status code」**만 보이면: (1) Edge 를 최신으로 `npm run deploy:functions`
     (2) **GitHub Pages 프론트도** 저장소 최신을 main 에 푸시해 Actions 로 다시 빌드·배포 (브라우저가 옛날 JS 를 쓰면
     오류 본문을 읽는 패치·Authorization 헤더가 반영되지 않을 수 있음).
   - 플레이NC HTML에 전투력·템렙 블록이 없으면(SPA만 렌더) 자동 갱신이 실패할 수 있습니다.
     그때는 표에서 수동 입력하거나 npm run aion2tool:power(아툴·Playwright)를 사용합니다.
