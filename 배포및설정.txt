아이온2 / 로스트아크 공용 일정 — GitHub Pages + Supabase 설정

1) Supabase (무료)
   - https://supabase.com 에서 프로젝트 생성
   - SQL Editor에서 supabase/schema.sql 내용 실행 (또는 migrations 순서대로 db push)
   - Authentication → Providers:
     - **Discord** 활성화 (Discord Developer Portal 에서 앱 생성 → Client ID / Client Secret)
     - Redirect URLs 에 실제 사이트 URL 등록:
       * 로컬: http://localhost:5173/ 및 http://127.0.0.1:5173/ (Vite 기본)
       * GitHub Pages: https://<계정>.github.io/<저장소>/ 및 필요 시 **/** 와일드카드 규칙에 맞게 추가
   - Project Settings → API 에서 URL / anon public key 복사

2) 로컬 개발
   - 프로젝트 루트에 .env 파일 생성:

     VITE_SUPABASE_URL=https://xxxx.supabase.co
     VITE_SUPABASE_ANON_KEY=eyJhbGciOi...

     (선택) 로스트아크 전용 입장 경로 — .env.example 참고
       VITE_LOSTARK_ENTRY_PATH=lostark
       또는 VITE_LOSTARK_ENTRY_HASH=#/lostark

   - npm install
   - npm run dev

3) GitHub Pages (Actions 빌드)
   - 저장소 Settings → Secrets and variables → Actions
     - VITE_SUPABASE_URL
     - VITE_SUPABASE_ANON_KEY
     - (선택) VITE_BASE
       * 기본값: /<저장소이름>/  (예: https://user.github.io/repo-name/)
       * <계정>.github.io 의 루트 저장소(사용자 페이지)처럼 경로가 '/' 여야 하면 VITE_BASE 를 "/" 로 지정
     - (선택) VITE_LOSTARK_ENTRY_PATH / VITE_LOSTARK_ENTRY_HASH — 로스트아크 탭 노출 조건
   - Settings → Pages → Build and deployment → Source: GitHub Actions
   - main(또는 master)에 푸시하면 자동 배포
   - **경로로 로스트아크 입장**을 쓰는 경우: `npm run build` 가 dist/index.html 을 dist/404.html 로 복사합니다.
     GitHub Pages 가 알 수 없는 하위 경로 요청 시에도 SPA 가 뜨도록 필요합니다.

4) 데이터 모델
   - raid_availability: user_id, raid_type(rudra | bagot | lostark), nickname, slots[], updated_at
   - 기존 DB에 server_name / combat_power 컬럼이 있으면 migrations 의 20260218120000 스크립트로 제거·raid_type 확장

5) 로스트아크 탭
   - VITE_LOSTARK_ENTRY_PATH 또는 VITE_LOSTARK_ENTRY_HASH 가 설정된 경우에만, 해당 주소로 들어온 브라우저에서
     헤더에「아이온2 | 로스트아크」전환이 보입니다. 비밀 경로를 아는 사람만 로스트아크 공용 슬롯(raid_type=lostark)에
     접근할 수 있습니다.
   - 둘 다 비우면 아이온2(루드라/바고트)만 제공됩니다.
