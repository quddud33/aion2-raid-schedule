1-1) Discord 로그인 — 리디렉션·콜백 (한국어 UI 기준)

   **개념**

   - 로그인 흐름: 브라우저 → Discord 동의 화면 → **Supabase 콜백 주소**로 돌아옴 → Supabase가 세션 처리 →

     마지막에 **우리 웹앱 주소**로 이동.

   - 그래서 **Discord 쪽**과 **Supabase 쪽**에 넣는 URL이 서로 다릅니다.



   **A. Discord 개발자 포털** (https://discord.com/developers/applications)

   - 왼쪽 **「기본 정보」**에 있는 **인터랙션 엔드포인트 URL**, **연결된 역할 인증 URL**, **이용 약관 URL**,

     **개인정보 보호 정책 URL** 은 OAuth 리디렉션이 **아닙니다.** (여기에 Supabase 콜백을 넣지 마세요.)

   - **애플리케이션** 선택 → 왼쪽 **OAuth2** → **일반**(General) 화면.

   - **리디렉션** / Redirects 목록에 **아래 한 줄을 반드시 추가**합니다.

     → Supabase가 “Discord용 OAuth 콜백 URL”이라고 알려주는 주소와 **동일**해야 합니다.

     형식:  https://<프로젝트-참조-ID>.supabase.co/auth/v1/callback

     (예: https://abcd1234.supabase.co/auth/v1/callback)

   - 이 주소는 **우리 GitHub Pages 주소가 아닙니다.** Discord는 “인증 결과를 Supabase로 넘긴다”고만 알면 됩니다.

   - **클라이언트 정보**: **클라이언트 ID**는 항상 보입니다. **클라이언트 시크릿**은 보안상 숨겨져 있으면

     **비밀 키 초기화** 후 새로 나온 값을 복사해 Supabase에 붙여넣습니다.



   **B. Supabase 대시보드** (프로젝트 열기)

   - 왼쪽 **인증**(Authentication) → **공급자**(Providers) → **Discord** 를 켭니다.

   - **클라이언트 ID** / **클라이언트 시크릿**에 Discord에서 복사한 값을 넣고 저장합니다.

   - 같은 화면(또는 Discord 설정 근처)에 **OAuth용 콜백 URL** / Callback URL 문구가 있으면,

     그 값이 위 A에서 Discord 리디렉션에 넣은 `.../auth/v1/callback` 과 같은지 확인합니다.



   **C. Supabase — “로그인 후 돌아올 웹앱 주소” (리디렉션 URL)**

   - 왼쪽 **인증** → **URL 구성**(URL Configuration) 메뉴로 이동합니다.

   - **사이트 URL**(Site URL): 배포된 사이트의 기본 주소 한 개.

     예) https://내계정.github.io/저장소이름/  (끝에 `/` 있어도 없어도 되나, 다른 항목과 통일)

   - **추가 리디렉션 URL** / Redirect URLs: OAuth 완료 후 **브라우저가 실제로 열릴 수 있는** 주소를 모두 넣습니다.

     이 목록에 없으면 로그인 후 “리디렉션이 허용되지 않음”류 오류가 납니다.

     예시)

       * http://localhost:5173/

       * http://127.0.0.1:5173/

       * https://내계정.github.io/저장소이름/

       * (로스트아크 전용 경로를 쓰는 경우) https://내계정.github.io/저장소이름/lostark

       * (해시 입장을 쓰는 경우) https://내계정.github.io/저장소이름/#/lostark

     ※ Supabase 플랜에 따라 와일드카드(`**` 등) 허용 여부가 다를 수 있어, 가능하면 **실제로 쓰는 URL을

       하나씩** 추가하는 것이 안전합니다.



   **D. 이 프로젝트 코드와 맞추기**

   - `App.tsx`의 Discord 로그인은 `redirectTo`로 **현재 주소 전체**(경로·해시 포함)를 넘깁니다.

   - 따라서 **C의 추가 리디렉션 URL**에, 개발·배포에서 실제로 주소창에 찍히는 형태와 맞는 항목이 있어야 합니다.

3) GitHub Pages (Actions 빌드)

   **GitHub에 시크릿 넣는 방법** (저장소 웹에서)

   1. GitHub에서 해당 **저장소** 페이지를 엽니다.

   2. 상단 **Settings**(설정) 탭을 클릭합니다.

   3. 왼쪽 메뉴 **Secrets and variables** → **Actions** 를 선택합니다.

   4. **New repository secret** 버튼으로 하나씩 추가합니다.

      - Name(이름) 칸과 Secret(값) 칸에 아래 표와 같이 넣습니다. 이름은 **대소문자·철자 그대로**여야 합니다.

      - 필수: `VITE_SUPABASE_URL` , `VITE_SUPABASE_ANON_KEY` (Supabase 대시보드에서 복사한 값)

      - 선택: `VITE_BASE` — 비우면 워크플로가 자동으로 `/<저장소이름>/` 를 씁니다. 사용자 페이지처럼 루트가 `/` 면

        Name `VITE_BASE` , Secret `/` (슬래시 하나)

      - 선택: `VITE_LOSTARK_ENTRY_PATH` — 예: Name 그대로, Secret 에 `lostark` 만 (따옴표 없이)

      - 선택: `VITE_LOSTARK_ENTRY_HASH` — 예: Secret 에 `#/lostark` 정확히

   5. 저장 후 **main**(또는 **master**)에 푸시하면 Actions 가 빌드할 때 위 값이 주입됩니다.

   (요약) Settings → Secrets and variables → Actions → New repository secret

     - VITE_SUPABASE_URL

     - VITE_SUPABASE_ANON_KEY

     - (선택) VITE_BASE

       * 기본값: /<저장소이름>/  (예: https://user.github.io/repo-name/)

       * <계정>.github.io 의 루트 저장소(사용자 페이지)처럼 경로가 '/' 여야 하면 VITE_BASE 를 "/" 로 지정

     - (선택) VITE_LOSTARK_ENTRY_PATH / VITE_LOSTARK_ENTRY_HASH — 로스트아크 탭 노출 조건

   - Settings → Pages → Build and deployment → Source: GitHub Actions

   - main(또는 master)에 푸시하면 자동 배포

   - **경로로 로스트아크 입장**을 쓰는 경우: `npm run build` 가 dist/index.html 을 dist/404.html 로 복사합니다.

     GitHub Pages 가 알 수 없는 하위 경로 요청 시에도 SPA 가 뜨도록 필요합니다.



4) 데이터 모델

   - raid_availability: user_id, raid_type(rudra | bagot | lostark), nickname, avatar_url(프로필 이미지 URL), slots[], updated_at

5) 로스트아크 탭 — 어떻게 접근하나요?

   - 빌드·배포 시 `.env` 또는 GitHub Actions 시크릿에 **둘 중 하나 이상**을 넣습니다. (`.env.example` 참고)

       * `VITE_LOSTARK_ENTRY_PATH=lostark`  → Vite `base` 뒤 **첫 경로**가 `lostark`일 때만 탭 노출

       * 또는 `VITE_LOSTARK_ENTRY_HASH=#/lostark`  → 주소의 **해시**가 정확히 `#/lostark`일 때만 탭 노출

   - **경로로 입장** 예: `https://내계정.github.io/저장소이름/lostark` (로컬은 `http://localhost:5173/lostark`)

   - **해시로 입장** 예: `https://내계정.github.io/저장소이름/#/lostark`

   - 위 조건으로 들어온 브라우저에서만 헤더에「아이온2 | 로스트아크」전환이 보입니다. 로스트아크 쪽 화면에서는

     하위 레이드 **에기르 나메** 일정을 맞추며, DB 저장 값은 `raid_type=lostark` 입니다.

   - `VITE_LOSTARK_ENTRY_PATH` 와 `VITE_LOSTARK_ENTRY_HASH` 를 **둘 다 비우면** 아이온2(루드라/바고트)만 제공됩니다.
